第四章 网络层

传输层–报文段segments 网络层–数据报datagrams

每个主机、路由器都需要运行网络层协议

转发&路由

转发：一个接口到另一个接口

路由：整条传送电路

01 虚电路和数据网络

网络层：面向连接服务在端系统及网络核心的路由器中实现。

数据报-》无连接 ；虚电路-》连接 两者选其一

如果数据报的话，会影响扩展性，所以无因特网无网络层的面向连接

虚电路 类似于电话电路

一条虚电路包括（vc)：

  • 一条从发送方到接收方的路径
  • vc号，每条链路都有一个vc号
  • 沿路径的每个路由器上都有转发表条目

适用于：用于ATM, frame-relay, X.25网络 。不能用于目前的因特网

数据报

数据报转发表，范围内口的转发口

存在方法：最长前缀匹配

数据报转发表会随着选入算法更新（1~5分钟；虚电路随着建立和拆除

02 路由器工作原理

路由器功能：路由算法；正确转发

  1. 输入端口功能

    • 线路端接模块：物理链路接到路由器的物理层

    • 数据链路处理模块：实现数据链路层功能

    • 查找与转发模块：以便分组通过路由器交换结构转发到适当的输出端口

      分布式交换：

      ​ 在每个输入端口本地做出交换决策，无需激活中央处理器

      ​ 目的：以线束速完成输入端口的处理

      ​ 排队：到达速度快于输出速度

  2. 交换结构：

    • 经内存的交换结构：经cpu，类似于io，拷贝缓存，受带宽
    • 经总线的交换结构：总线速率，每次仅一个分组
    • 经交换矩阵的交换结构：高于其他两个

  3. 输出端口功能

    ​ 同输入端，对称的物理结构

数据报-》数据帧（链路

线头阻塞：队头阻塞，后面都要等候

05 层次选路

整个网络并非平面，所以不可能完整记录全部

AS：一个区域内的路由器组成的集合 “自治系统”

同一个AS中的路由协议为：区域内路由协议IGP；与其他AS：域间路由协议

  • 区域内路由协议IGP*

    标准的域内路由协议:
    RIP: 路由信息协议Routing Information Protocol

    每隔30s，通过响应报文在邻居间交换距离向量（也称RIP通告

    若180秒后没有收到通告，则认为邻居死机或链路中断：发公告、毒性逆转

  • OSPF (Open Shortest Path First)

    用链路状态算法：分发LS 分组、每个节点具有拓扑图、使用 Dijkstra算法

    通告会散布到 整个 自治系统 (通过洪泛法)
    OSPF信息直接通过 IP传输 (不是 TCP 或 UDP）

  • Internet 域间选路: BGP

    BGP:Border Gateway Protocol

    从相邻AS获取子网可达信息
    向该AS内部的所有路由器传播这些可达性信息

    路由表由AS内和AS间算法构成

    • AS域间任务：

      AS1传给域外的分组，需要知道相连的AS2和AS3能到达哪些目的端，并将这些信息传播给AS1内所有路由器

      注意：如果AS1到达目的地，AS2和AS3均可，直接*热土豆算法*，选择通向最低费用网关的接口I,

      采用TCP来交换报文
